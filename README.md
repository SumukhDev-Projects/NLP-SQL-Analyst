# NL-SQL Analyst

A natural language interface to a SQL database, powered by Claude. Type a question in plain English, get back the SQL, the results, and a chart вЂ” all in one shot.

The core problem this solves: most data analysts spend a disproportionate amount of time writing boilerplate SQL for straightforward lookups. This tool lets you ask "which product categories generated the most revenue last quarter?" and get an answer in seconds, with the generated SQL visible so you can inspect, copy, or modify it.

Built this to demonstrate how LLMs can be a practical layer on top of structured data вЂ” not just a chatbot, but something that actually touches a database and returns verifiable results.

---

## How it works

1. You type a question in the input box (or switch to SQL mode and write the query directly).
2. The backend sends the full database schema plus your question to Claude.
3. Claude returns a SELECT statement.
4. The backend validates the SQL (read-only enforcement, injection prevention), executes it against a local SQLite database, and returns the results.
5. The frontend renders a results table and, when the data shape allows it, automatically generates a chart.

The generated SQL is always shown alongside the results, so you can see exactly what ran and verify it makes sense for the question asked.

## Sample database

The project ships with a pre-seeded SQLite database containing synthetic but realistic sales data:

| Table | Rows | Description |
|---|---|---|
| orders | ~3,500 | Order headers with date, status, total amount |
| order_items | ~6,000 | Line items per order with quantity, price, discount |
| customers | 500 | Customer profiles with segment, location, signup date |
| products | 23 | Product catalog with category, price, cost |
| sales_reps | 20 | Sales reps with region and annual quota |

This gives enough relational depth to produce interesting queries вЂ” joins, aggregations, time series, ranking, segment comparisons.

## Stack

**Backend**

- FastAPI with a synchronous SQLite connection via the standard `sqlite3` library
- All queries run in read-only mode (`mode=ro` URI + `PRAGMA query_only = ON`)
- SQL safety layer rejects anything that isn't a SELECT or WITH statement
- Claude (`claude-opus-4-5`) handles NL-to-SQL translation; the full schema is passed as context on every request
- pandas for query execution and result serialisation
- Chart configuration generated server-side based on result shape (Plotly-compatible JSON)

**Frontend**

- React + Vite + Tailwind
- Two input modes: natural language and raw SQL editor
- Results panel with table view and chart view (Plotly.js rendered client-side)
- Schema explorer sidebar with column types and row counts
- Query history stored in component state for the session

## Getting started

### Prerequisites

- Docker Desktop, or Python 3.11+ and Node 20+
- Anthropic API key (from console.anthropic.com)

### With Docker

```bash
git clone https://github.com/YOUR_USERNAME/nl-sql-analyst
cd nl-sql-analyst

cp .env.example .env
# Add your ANTHROPIC_API_KEY to .env

make up
```

Open http://localhost:3000.

The database is seeded automatically on first run. FastAPI docs are at http://localhost:8000/docs.

### Without Docker

```bash
# 1. Seed the database (one-time)
make install-backend
make seed

# 2. Run the backend
make run-backend
# в†’ http://localhost:8000

# 3. Run the frontend (separate terminal)
make install-frontend
make run-frontend
# в†’ http://localhost:5173
```

## API

Full docs at http://localhost:8000/docs.

```bash
# Natural language query
curl -X POST http://localhost:8000/api/query \
  -H "Content-Type: application/json" \
  -d '{"question": "What are the top 5 customers by revenue in 2023?"}'

# Raw SQL (read-only)
curl -X POST http://localhost:8000/api/sql \
  -H "Content-Type: application/json" \
  -d '{"sql": "SELECT category, SUM(total_amount) AS revenue FROM orders JOIN order_items oi ON oi.order_id = orders.order_id JOIN products p ON p.product_id = oi.product_id GROUP BY category ORDER BY revenue DESC"}'

# Schema
curl http://localhost:8000/api/schema

# Sample questions
curl http://localhost:8000/api/suggestions
```

**Query response shape:**

```json
{
  "sql": "SELECT ...",
  "columns": ["customer", "revenue"],
  "rows": [["Acme Corp", 48200.0], ...],
  "row_count": 5,
  "truncated": false,
  "chart": { "type": "bar", "data": [...], "layout": {...} },
  "execution_ms": 124,
  "error": null
}
```

## Security

The read-only enforcement has two layers. First, the SQLite connection opens the database with `mode=ro` in the URI and sets `PRAGMA query_only = ON`. Second, the `sql_safety.py` validator checks the query against a whitelist before execution:

- Only statements starting with SELECT or WITH (CTEs) are accepted
- A blocklist rejects DROP, DELETE, INSERT, UPDATE, CREATE, ALTER, TRUNCATE, ATTACH, PRAGMA and others
- Multiple statements separated by semicolons are rejected
- The backend never exposes raw exception details to the client

This makes it safe to expose a raw SQL input in the UI вЂ” the worst someone can do is run a slow or complex SELECT.

## Configuration

| Variable | Default | Description |
|---|---|---|
| `ANTHROPIC_API_KEY` | required | Your Anthropic API key |
| `CLAUDE_MODEL` | `claude-opus-4-5` | Claude model to use |
| `MAX_ROWS_RETURNED` | `500` | Hard cap on result rows |
| `QUERY_TIMEOUT_SECONDS` | `30` | SQLite query timeout |
| `DEBUG` | `false` | Enable FastAPI debug mode |

## Extending the database

To use your own data instead of the sample dataset:

1. Add your SQLite database to `backend/data/`
2. Update `DB_PATH` in `.env` to point to your file
3. The schema explorer and Claude's context will automatically reflect the new tables

If you're connecting to PostgreSQL or another database, swap out `backend/app/db/connection.py` вЂ” the rest of the stack doesn't care what's underneath.
